# 기획 규칙

계획, 설계 단계에 적용되는 공통 규칙. 코드 작업뿐 아니라 콘텐츠 기획, 카드뉴스, 문서 설계 등 모든 "만들기 전 결정" 단계에 해당된다.

## 디폴트로 grill-with-docs 발동

**Why:** 사용자(Lee Ji-han)는 실행 전에 결정 트리의 모든 가지를 한 번에 하나씩 짚어가며 의사결정을 명확히 해두는 흐름을 선호한다. 결정 확정 시 곧바로 `decisions/` (ADR) 또는 `CONTEXT.md` 에 박히는 흐름이 본인의 records-first 운영 방식과 맞물린다.

**How to apply:**

### 자동 발동 트리거 (디폴트 ON)

다음 중 하나라도 해당되면, **답하기 전에 `grill-with-docs` 스킬을 명시적으로 invoke** 한다. 스킬이 없는 환경이면 같은 정신 — "한 번에 하나씩 질문, 매번 추천 답 제시, 결정 시 문서 인라인 업데이트" — 을 수동으로 적용한다.

- 새 기능, 모듈, 디렉토리 추가 요청 ("X 만들어줘", "Y 구현해줘")
- 여러 파일에 걸친 변경, 리팩토링, 아키텍처 결정
- 라이브러리, 데이터 모델, API 형태 선택
- PRD/스펙/플랜 작성 요청
- 콘텐츠 기획 (카드뉴스 시리즈, 영상 커리큘럼 등)
- "어떻게 하지", "어떤 방식이 좋을까" 같이 의사결정을 묻는 질문

### 건너뛰기 (자동 발동 X)

- 단일 파일 단순 수정 (오타, 변수명, 작은 버그 수정)
- 명시적 일회성 명령 ("이 명령 실행", "이 파일 읽어", "git status 보여줘")
- 사용자가 의사결정을 이미 끝낸 상태에서 손만 빌리는 작업
- `/scribe` 같은 자동화, 정해진 워크플로 트리거

### 명시적 skip 신호

사용자가 다음 중 하나를 말하면 grill 을 건너뛰고 바로 실행한다.

- "그냥 해줘"
- "skip grill" / "grill 빼고"
- "질문 말고"
- "바로 가자" / "바로 실행"
- "내가 다 정했어"

### 폴백 — `grill-with-docs` 가 없는 환경

레포에 `CONTEXT.md` 나 `decisions/` (또는 `docs/adr/`) 가 없으면 `grill-me` 수준으로 작동 — 도메인 문서 정렬 단계를 생략하고 결정만 한 번에 하나씩 짚는다. 결정 결과는 채팅에 정리해두고, 사용자가 "문서로 박아" 라고 하면 그때 ADR 생성.
