# 기획 규칙

계획, 설계 단계 공통. 코드뿐 아니라 콘텐츠 기획, 카드뉴스, 문서 설계 등 "만들기 전 결정" 단계 전부.

## grill-with-docs 디폴트 발동

**Why:** 사용자(Lee Ji-han)는 실행 전 결정 트리의 모든 가지를 한 번에 하나씩 짚어 확정하는 흐름을 선호. 확정 즉시 `decisions/` (ADR) 나 `CONTEXT.md` 에 반영하는 records-first 방식과 맞물린다.

### 자동 발동 (디폴트 ON)

아래 중 하나면 **답하기 전에 `grill-with-docs` 스킬을 invoke**. 스킬이 없으면 같은 정신("한 번에 하나씩 질문, 매번 추천 답 제시, 결정 시 문서 인라인 업데이트")을 수동 적용.

- 새 기능, 모듈, 디렉토리 추가 ("X 만들어줘", "Y 구현해줘")
- 여러 파일 변경, 리팩토링, 아키텍처 결정
- 라이브러리, 데이터 모델, API 형태 선택
- PRD/스펙/플랜 작성
- 콘텐츠 기획 (카드뉴스 시리즈, 영상 커리큘럼 등)
- "어떻게 하지", "어떤 방식이 좋을까" 같은 의사결정 질문

### 건너뛰기

- 단일 파일 단순 수정 (오타, 변수명, 작은 버그)
- 일회성 명령 ("이 명령 실행", "이 파일 읽어", "git status")
- 사용자가 이미 결정한 상태에서 손만 빌리는 작업
- `/scribe` 같은 자동화, 정해진 워크플로

### 명시적 skip 신호

"그냥 해줘", "skip grill" / "grill 빼고", "질문 말고", "바로 가자" / "바로 실행", "내가 다 정했어" 중 하나면 grill 건너뛰고 바로 실행.

### 폴백 (스킬 없는 환경)

레포에 `CONTEXT.md` 나 `decisions/` (`docs/adr/`) 가 없으면 `grill-me` 수준 — 도메인 문서 정렬은 생략하고 결정만 하나씩 짚는다. 결과는 채팅에 정리하고, 사용자가 "문서로 남겨" 라 하면 그때 ADR 생성.
