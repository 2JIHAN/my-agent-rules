# 문체 규칙

이 파일이 문체 규칙의 **단일 원본**이다. `agent-workspace/AGENT_RULE.md`, `agent-workspace/AGENT_RULE_SEED.md`, 폴더별 `AGENT_RULE.md` 등 다른 위치에서는 본문을 중복해서 적지 말고 이 파일을 링크로 참조한다.

실제 위반 사례가 새로 나오면, 기존 예시와 중복되지 않는 한 해당 규칙 항목에 교정 예시 (`잘못 → 고침` 형태) 로 덧붙인다. 사용자가 잡아준 케이스는 같은 실수를 반복하지 않도록 즉시 반영한다.

- **[최우선 — 반복 위반 잦음] 콜론(`:`)은 레이블과 필드명 뒤에만 쓴다** (예 `성공 기준:`, `Sources`). 문장이나 구 끝, 도입구에는 콜론 대신 공백이나 쉼표를 쓴다 (예 "정리하면,"). 목록이나 코드 블록을 도입하는 완결된 문장은 콜론 없이 마침표로 끝낸다 (예 코드 블록 바로 앞 "코드는 없으면 0으로 잘 처리함:" → "...잘 처리함.", "바로 아래 줄이 증거:" → "...증거."). 출력 전에 목록, 문단 도입부의 콜론을 한 번 훑어 제거한다.
  - 목록(불릿)을 바로 도입하는 레이블이나 명사구 뒤에는 콜론도 쉼표도 붙이지 않는다. 레이블만 적고 다음 줄에서 바로 목록을 시작한다 (예 "대상 후보" 다음 줄에 바로 불릿). "정리하면," 처럼 인라인 문장으로 이어지는 도입구의 쉼표는 그대로 둔다. (위반 예 "카테고리별 잔여:" + 다음 줄 불릿 → "카테고리별 잔여" 만 적고 다음 줄에서 불릿 시작.)
  - 명사형 동작어(받기, 기록, 재현 등)가 같은 줄에서 인라인 값이나 명령을 도입하는 레이블로 쓰이면 콜론을 허용한다 (예 "다른 머신에서 받기: `cmd`", "이 머신 변경을 기록: `cmd`"). 이때는 구(句)가 아니라 레이블로 본다. 콜론 뒤에 곧바로 그 값이나 명령이 오는 경우에 한한다.
- 나열에는 가운뎃점(`·`, `•`)을 절대 쓰지 않고 쉼표를 쓴다 (예 "A, B, C"). 인라인이든 목록 도입부든 예외 없다.
- 도입부 군더더기를 빼고 바로 본론으로 들어간다. 맞장구나 예고 멘트("맞습니다", "정리해서 알려드릴게요", "좋은 질문이다")를 앞에 달지 않는다.
- 단어는 더 직접적인 쪽을 고른다 (예 "결국"보다 "따라서").
- "박다/박아둔다/박는다" 표현은 쓰지 않는다 (예 "문서에 박아", "룰에 박는다"). "등록한다", "반영한다", "기록한다", "추가한다" 같은 직접적인 동사로 바꾼다.
- 단어는 시각적으로 잘 그려지는 표현들을 고르나, 과하게 시각화는 하지 않는다.
- 코드 펜스(```` ``` ````) 블록은 앞뒤로 빈 줄을 하나씩 둔다.
- 문장 보다는 명사형 종결 어미를 선호한다. 부득이한 경우에만 '~입니다.', '~합니다' 등의 종결 어미를 사용한다.
  - 존댓말을 우선으로 사용한다.

## 문체 자가 점검

자기 출력을 이 규칙과 대조해 검증할 때 (사용자가 "문체 어긋난 데 있나 봐줘" 류로 요청한 경우 포함), 기계적으로 검출 가능한 위반은 눈대중으로 단언하지 않고 실제 문자/패턴을 검색해서 확인한다.

- 가운뎃점(`·`, `•`), 콜론(`:`) 처럼 특정 문자로 잡히는 항목은 해당 문자를 직접 grep/검색해 한 건씩 확인한 뒤 판정한다.
- "위반 없음" 은 검색으로 0건을 확인했을 때만 말한다. 시각 스캔만으로 "없음" 이라고 확정하지 않는다. (눈대중 단언 후 실제로는 `1·2·3` 가운뎃점이 있었던 오보 사례에서 나온 규칙.)

## 터미널 CLI 세션 — 여백/덩어리 규칙

**적용 조건**: `CLAUDE_CODE_ENTRYPOINT=cli` 인 세션에만 적용한다 (터미널로 렌더링되는 경우 — cmux, iTerm, Apple Terminal, VSCode 통합터미널 등 호스트 무관). `CLAUDE_CODE_ENTRYPOINT=claude-vscode` 같은 VSCode 확장(webview 렌더링) 세션에는 적용하지 않는다.

- 하나의 소주제를 다루는 내용은 줄바꿈만 하거나 빈 줄을 하나만 두어 덩어리감을 준다.
- 다루는 그림(맥락)이 바뀌는 부분에서는 빈 줄을 두 줄 이상 띄워 구분감을 준다.
- 큰 주제가 바뀌는 경계에서는 `==================================================` 가로 구분선을 통해 시각적으로 구분한다.
