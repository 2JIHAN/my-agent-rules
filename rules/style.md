# 문체 규칙

이 파일이 문체 규칙의 **단일 원본**이다. `agent-workspace/AGENT_RULE.md`, `agent-workspace/AGENT_RULE_SEED.md`, 폴더별 `AGENT_RULE.md` 등 다른 위치에서는 본문을 중복해서 적지 말고 이 파일을 링크로 참조한다.

- 콜론(`:`)은 레이블과 필드명 뒤에만 쓴다 (예 `성공 기준:`, `Sources`). 문장이나 구 끝, 도입구에는 콜론 대신 공백이나 쉼표를 쓴다 (예 "정리하면,"). 목록이나 코드 블록을 도입하는 완결된 문장은 콜론 없이 마침표로 끝낸다.
  - 목록(불릿)을 바로 도입하는 레이블이나 명사구 뒤에는 콜론도 쉼표도 붙이지 않는다. 레이블만 적고 다음 줄에서 바로 목록을 시작한다 (예 "대상 후보" 다음 줄에 바로 불릿). "정리하면," 처럼 인라인 문장으로 이어지는 도입구의 쉼표는 그대로 둔다.
- 나열에는 가운뎃점(`·`, `•`)을 절대 쓰지 않고 쉼표를 쓴다 (예 "A, B, C"). 인라인이든 목록 도입부든 예외 없다.
- 도입부 군더더기를 빼고 바로 본론으로 들어간다. 맞장구나 예고 멘트("맞습니다", "정리해서 알려드릴게요", "좋은 질문이다")를 앞에 달지 않는다.
- 단어는 더 직접적인 쪽을 고른다 (예 "결국"보다 "따라서").
- 단어는 시각적으로 잘 그려지는 표현들을 고르나, 과하게 시각화는 하지 않는다.
- 코드 펜스(```` ``` ````) 블록은 앞뒤로 빈 줄을 하나씩 둔다.
- 문장 보다는 명사형 종결 어미를 선호한다. 부득이한 경우에만 '~입니다.', '~합니다' 등의 종결 어미를 사용한다.
  - 존댓말을 우선으로 사용한다.

## 터미널 CLI 세션 — 여백/덩어리 규칙

**적용 조건**: `CLAUDE_CODE_ENTRYPOINT=cli` 인 세션에만 적용한다 (터미널로 렌더링되는 경우 — cmux, iTerm, Apple Terminal, VSCode 통합터미널 등 호스트 무관). `CLAUDE_CODE_ENTRYPOINT=claude-vscode` 같은 VSCode 확장(webview 렌더링) 세션에는 적용하지 않는다.

- 하나의 소주제를 다루는 내용은 줄바꿈만 하거나 빈 줄을 하나만 두어 덩어리감을 준다.
- 다루는 그림(맥락)이 바뀌는 부분에서는 빈 줄을 두 줄 이상 띄워 구분감을 준다.
- 큰 주제가 바뀌는 경계에서는 `========================================` 가로 구분선을 통해 시각적으로 구분한다.
